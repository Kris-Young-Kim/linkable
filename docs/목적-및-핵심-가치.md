## 1. 프로젝트 목적

### 1.1 문제 해결 목적

LinkAble은 **정보의 비대칭으로 인해 자신에게 맞는 보조기기를 찾지 못하는 사용자들**을 위해 탄생했습니다.

#### 해결하고자 하는 문제들

1. **정보 탐색의 어려움**
   - 일반 쇼핑몰에는 수만 개의 보조기기가 존재하나, 분류 기준이 모호함
   - 사용자는 자신의 신체적 기능 저하(ICF-b)를 기기의 기술적 사양(ISO Code)으로 번역할 능력이 없음
   - 예: "손이 떨린다"는 알지만, "무게감 있는 식사 도구"를 검색해야 한다는 것을 모름

2. **전문가 접근성 제한**
   - 보조공학사나 작업치료사의 1:1 상담을 받으려면 복지관을 방문하거나 긴 대기 시간을 거쳐야 함
   - 급성기 이후 재가(Home)로 복귀한 환자들이 '골든타임'에 적절한 환경 수정을 받지 못함

3. **데이터 기반 검증 부재**
   - 기존 시장은 '판매량'이나 '단순 별점' 위주로 돌아감
   - "이 기기가 내 삶을 얼마나 변화시켰는가?"에 대한 기능적 효과성 데이터가 전무

### 1.2 서비스 목적

LinkAble은 단순한 쇼핑몰이 아닙니다. **16년 경력의 보조공학사/작업치료사의 임상 노하우를 AI에 학습시켜**, 사용자의 **작업 수행(Occupational Performance)**을 분석하고 최적의 도구를 매칭하는 **'디지털 코디네이터' 서비스**입니다.

### 1.3 개발 목표 (MVP Goals)

1. **접근성 극대화**
   - 텍스트 입력이 어려운 고령자를 위한 음성(STT) 및 이미지 기반 상담 구현

2. **데이터 구조화**
   - 비정형 상담 데이터를 ICF Code와 ISO Code로 자동 변환하는 파이프라인 구축

3. **검증 루프 완성**
   - 단순 구매에서 끝나는 것이 아니라, 사용 후 평가(K-IPPA) 데이터가 다시 AI 학습으로 이어지는 선순환 구조 확보

## 2. 핵심 가치

### 2.1 Professional (전문성)

**ICF(국제기능분류) 기준의 정밀한 기능 분석**

- 16년 경력의 보조공학 전문가의 임상 노하우를 AI에 학습
- 사용자의 자연어 호소를 ICF 코드로 구조화하여 정확한 문제 파악
- 신체 기능(b), 활동(d), 참여(p), 환경 요소(e)를 체계적으로 분석

### 2.2 Standardized (표준화)

**ISO 9999 국제 표준에 따른 보조기기 매칭**

- 국제 표준 ISO 9999에 기반한 보조기기 분류 및 매칭
- 추출된 ICF 코드를 ISO 코드로 자동 변환
- 표준화된 분류 체계로 일관성 있는 추천 제공

### 2.3 Evidence-based (근거 기반)

**K-IPPA 평가 도구를 통한 사용 후 효과성 데이터 검증**

- 구매 후 실제 사용 경험을 K-IPPA 평가 도구로 측정
- 사용 전/후 수행 난이도 변화를 정량적으로 분석
- 축적된 데이터를 AI 학습에 활용하여 추천 정확도 향상

### 2.4 Safe (안전성)

**의료법을 준수하는 비의료적 기능 보조 가이드라인 적용**

- 의학적 진단/처방 용어 절대 배제
- 기능적(Functional) 해결책만 제시
- 의료 행위를 기술적으로 차단하는 가드레일 적용

## 3. 비전 및 미션

### 3.1 비전 (Vision)

**"불편함을 가능성으로 연결하다 (Connecting Disabilities to Abilities)"**

LinkAble은 사용자의 불편함을 단순히 해결하는 것을 넘어, **가능성으로 전환**하는 것을 목표로 합니다. 기술과 사람을 잇는 따뜻한 연결고리가 되어, 모든 사용자가 자신의 삶을 더 풍요롭게 살아갈 수 있도록 돕습니다.

### 3.2 미션 (Mission)

**전주기적 보조공학 코디네이팅 서비스 제공**

- 사용자의 자연어 호소를 ICF 기준으로 구조화
- ISO 9999 표준 보조기기를 매칭
- K-IPPA를 통해 효과성을 검증
- 데이터 기반 선순환 구조로 지속적인 개선

### 3.3 목표 사용자 경험

1. **접근성**: 누구나 쉽게 사용할 수 있는 인터페이스
2. **전문성**: 전문가 수준의 정확한 분석과 추천
3. **신뢰성**: 표준화된 기준과 근거 기반 추천
4. **지속성**: 구매 후 효과성 검증까지 이어지는 완전한 경험

## 4. 핵심 원칙

### 4.1 Non-Medical (비의료적)

**의학적 진단/처방 용어 절대 배제 → 기능적(Functional) 해결책 제시**

- '링커'는 의사가 아니며, 의학적 진단이나 처방을 제공하지 않습니다
- 오직 사용자의 **기능적 불편함(Functional Limitation)**을 보완하기 위한 보조공학적 정보만을 제공합니다
- 모든 추천 결과는 참고용이며, 건강상의 중대한 문제는 반드시 전문 의료인과 상의해야 합니다

### 4.2 Evidence-based (근거 기반)

**모든 추천은 임상적 근거(ICF, ISO)와 데이터(IPPA)에 기반함**

- ICF(국제기능분류) 기준의 체계적인 문제 분석
- ISO 9999 국제 표준에 따른 보조기기 매칭
- K-IPPA 평가 데이터를 통한 효과성 검증
- 축적된 데이터를 AI 학습에 활용하여 지속적인 개선

### 4.3 Accessibility First (접근성 우선)

**WCAG 2.1 AA 표준 준수 (필수)**

- 키보드 네비게이션: 모든 인터랙션은 Tab 키로 이동 가능
- 스크린 리더 지원: 모든 이미지에 alt 텍스트, 버튼과 링크에 aria-label
- 색상 대비: WCAG 2.1 AA 기준 준수 (최소 4.5:1)
- 포커스 표시: 모든 포커스 가능 요소에 명확한 포커스 링
- 텍스트 크기: rem 단위 사용으로 브라우저 폰트 크기 설정 존중

### 4.4 User-Centered (사용자 중심)

**사용자의 경험과 니즈를 최우선으로 고려**

- 멀티모달 입력: 텍스트, 음성, 이미지 등 다양한 입력 방식 지원
- 직관적인 인터페이스: 복잡한 전문 용어 대신 일상 언어 사용
- 개인화된 추천: 사용자의 신체적 제약과 환경을 고려한 맞춤형 추천
- 지속적인 피드백: K-IPPA 평가를 통한 사용자 경험 개선

### 4.5 Data-Driven (데이터 기반)

**데이터를 통한 지속적인 개선과 검증**

- 상담 데이터 수집 및 분석
- 추천 정확도 추적
- K-IPPA 평가 데이터 축적
- AI 학습을 통한 추천 정확도 향상

## 5. 방법론: 활동 분석 (Activity Analysis Approach)

LinkAble은 사용자의 경험을 작업치료의 **활동 분석(Activity Analysis)** 프레임워크에 따라 4단계로 세분화하여 설계되었습니다.

| 단계 | 활동 (Activity) | 시스템 로직 (System Logic) |
|------|-----------------|---------------------------|
| Step 1 | 문제 호소 (Assessment) | 자연어/음성 입력을 통해 신체 기능(b), 활동(d), 환경(e) 코드를 추출합니다 |
| Step 2 | 해결책 탐색 (Matching) | 추출된 문제(Problem)와 해결 도구(Solution)의 Gap을 분석하여 ISO 코드를 매핑합니다 |
| Step 3 | 구매 실행 (Action) | 매칭된 ISO 코드를 가진 최적의 상품을 E-commerce로 연결합니다 |
| Step 4 | 가치 검증 (Validation) | 사용 전/후 수행 난이도 변화를 측정하여 K-IPPA 효과성 점수를 산출합니다 |

## 6. 핵심 가치의 실현 방법

### 6.1 Professional 실현

- **전문가 로직 AI화**: 16년 경력 전문가의 임상 노하우를 시스템 프롬프트에 반영
- **ICF 코드 추출**: 자연어 입력을 ICF 코드로 자동 변환하는 정밀한 파싱 로직
- **체계적 분석**: 신체 기능, 활동, 환경 요소를 분리하여 구조화된 분석 제공

### 6.2 Standardized 실현

- **ISO 9999 매핑**: ICF 코드를 ISO 코드로 자동 변환하는 매핑 테이블
- **표준 분류 체계**: 국제 표준에 따른 일관성 있는 보조기기 분류
- **검증 가능한 추천**: 표준화된 기준으로 추천 근거를 명확히 제시

### 6.3 Evidence-based 실현

- **K-IPPA 평가 시스템**: 사용 전/후 점수와 중요도를 측정하는 평가 도구
- **효과성 데이터 축적**: 평가 데이터를 데이터베이스에 저장하여 추적
- **데이터 기반 개선**: 축적된 데이터를 AI 학습에 활용하여 추천 정확도 향상

### 6.4 Safe 실현

- **시스템 프롬프트 가드레일**: 의료 행위를 기술적으로 차단하는 프롬프트 설정
- **기능 중심 언어**: 의료 용어 대신 기능 중심의 일상 언어 사용
- **명확한 면책 고지**: 모든 추천 결과에 참고용임을 명시

## 7. 지속 가능한 가치 창출

### 7.1 사용자 가치

- **시간 절약**: 복잡한 검색 과정 없이 빠른 추천 받기
- **정확한 매칭**: 전문가 수준의 정확한 보조기기 추천
- **효과성 검증**: 실제 사용 경험을 통한 효과성 확인

### 7.2 사회적 가치

- **접근성 향상**: 누구나 전문가 수준의 보조공학 서비스 이용 가능
- **데이터 축적**: 보조기기 효과성에 대한 실증 데이터 구축
- **정책 지원 근거**: 정부나 보험사의 지원 근거 마련에 기여

### 7.3 비즈니스 가치

- **지속 가능한 수익 모델**: 제휴 수수료, 데이터 비즈니스, B2B SaaS
- **확장 가능한 구조**: AI 학습을 통한 지속적인 개선
- **시장 선점**: 초고령 사회 진입에 따른 시장 기회 활용
