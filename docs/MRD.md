Advanced MRD (Market Requirements Document)
프로젝트명: LinkAble (링케이블)
작성자: 16년 차 보조공학 전문가 / 에이블 코디 팀
버전: v2.0 (Activity Analysis Based)

1. 시장 문제 정의 (Market Problem) : 활동 분석 관점
   우리는 시장의 문제를 단순히 '불편함'이 아닌, 사용자의 작업 수행(Occupational Performance) 단절 관점에서 분석합니다.
   1.1 정보 탐색의 인지적 부하 (Cognitive Load in Discovery)
   현상: 일반 쇼핑몰(쿠팡, 네이버)에는 수만 개의 보조기기가 존재하나, 분류 기준이 모호함.
   활동 분석:
   사용자는 자신의 신체적 기능 저하(ICF-b)를 기기의 기술적 사양(ISO Code)으로 번역할 능력이 없음.
   예: "손이 떨린다(Tremor)"는 알지만, "Weighted Utensil(무게감 있는 식사 도구)"를 검색해야 한다는 것을 모름.
   Pain Point: 검색 실패 -> 부적절한 기기 구매 -> 사용 포기 -> 기능 저하의 악순환.
   1.2 전문가 접근성 제한 (Limited Professional Access)
   현상: 보조공학사나 작업치료사의 1:1 상담을 받으려면 복지관을 방문하거나 긴 대기 시간을 거쳐야 함.
   Pain Point: 급성기 이후 재가(Home)로 복귀한 환자들이 '골든타임'에 적절한 환경 수정을 받지 못함.
   1.3 데이터 기반 검증 부재 (Lack of Evidence)
   현상: 기존 시장은 '판매량'이나 '단순 별점' 위주로 돌아감.
   Pain Point: "이 기기가 내 삶을 얼마나 변화시켰는가?"에 대한 기능적 효과성(K-IPPA) 데이터가 전무하여, 정부나 보험사의 지원 근거가 약함.
2. 타겟 시장 및 페르소나 (Target Market & Personas)
   인구통계학적 분류를 넘어, **'기술 수용도'와 '신체 기능 수준'**에 따라 타깃을 세분화합니다.
   2.1 Primary Target (핵심 수요자)
   세그먼트: 고령자(65+) 및 경증 장애인 당사자, 그들의 자녀(3040 보호자).
   시장 규모: 대한민국 초고령 사회 진입(2025년 인구의 20%가 노인), 실버 이코노미 시장 약 168조 원 규모.
   페르소나 A (김철수 님, 72세):
   상태: 파킨슨 초기, 스마트폰 카카오톡 정도만 사용 가능.
   Needs: 복잡한 검색 없이 말(Voice)로 하면 알아서 찾아주는 "비서"가 필요.
   페르소나 B (이영희 님, 42세 딸):
   상태: 직장인, 떨어져 사는 어머니 걱정.
   Needs: 전문가가 검증해 준 "실패 없는 제품"을 빨리 보내드리고 싶음 (효도 소비).
   2.2 Secondary Target (B2B 파트너)
   세그먼트: 복지용구 사업소, 요양보호사 교육기관.
   Needs: 상담 시간을 단축하고, 고객에게 근거 있는 추천을 해줄 수 있는 CRM 도구 필요.
3. 솔루션 가치 제안 (Unique Value Proposition)
   "LinkAble은 무엇이 다른가?"
   구분 기존 오픈마켓 (Coupang) 오프라인 센터 (Assistive Tech Center) LinkAble (MVP)
   접근성 높음 (앱) 낮음 (방문 필요) 매우 높음 (AI 챗봇)
   전문성 낮음 (리뷰 의존) 높음 (전문가 상주) 높음 (전문가 로직 AI)
   추천 기준 인기순/광고순 임상적 판단 ICF + ISO 표준 매칭
   사후 관리 반품/환불 제한적 K-IPPA 효과성 검증
4. 비즈니스 모델 및 수익화 전략 (Business Model)
   MVP 단계에서는 **'트래픽 확보'와 '데이터 축적'**을 최우선으로 하되, 명확한 현금 흐름(Cash Flow)을 만듭니다.
   4.1 1단계: 중개 및 제휴 (Affiliate Model) - MVP
   구조: AI 추천 결과에서 '구매하기' 클릭 시 쿠팡/네이버/케어라인 등으로 연결.
   수익: 구매 전환 발생 시 판매액의 3~5% 수수료(Affiliate Link) 수취.
   장점: 재고 부담 없음, 초기 개발 비용 최소화.
   4.2 2단계: 데이터 비즈니스 (Data Monetization) - Scale-up
   구조: K-IPPA 평가 데이터(제품별 효과성 리포트)를 제조사나 보험사에 판매.
   가치: "어떤 증상의 노인에게 어떤 기기가 가장 효과적인가?"에 대한 RWE(Real World Evidence) 확보.
   4.3 3단계: B2B SaaS (Subscription)
   구조: 복지용구 사업소에 'AI 상담 솔루션'을 월 구독 형태로 제공.
5. 시장 진입 전략 (Go-to-Market Strategy)
   5.1 초기 확산 (Seed Phase)
   커뮤니티 공략: 뇌졸중 환우회, 파킨슨병 카페, 노인장기요양보호사 커뮤니티에 "무료로 보조기기 컨설팅해 드립니다" 바이럴 마케팅.
   전문가 브랜딩: 16년 차 전문가의 블로그/유튜브와 연계하여 신뢰도 확보.
   5.2 성장 (Growth Phase)
   SEO 최적화: "할머니 지팡이 추천", "편마비 식사 도구" 등 롱테일 키워드 검색 시 LinkAble의 AI 상담 페이지가 노출되도록 설정 (Next.js SSR 활용).
6. 핵심 성공 지표 (KPIs)
   Consultation Completion Rate: AI 상담 완료율 (목표: 60% 이상).
   Recommendation Click-through Rate (CTR): 추천 상품 클릭률 (목표: 15% 이상).
   K-IPPA Feedback Rate: 구매 후 평가 참여율 (목표: 10% 이상 - 포인트 보상 활용).
   💡 전문가의 코멘트 (For Developers)
   이 문서는 개발팀에게 **"우리가 단순히 쇼핑몰을 만드는 게 아니라, '디지털 헬스케어의 빈틈'을 메우는 혁신을 하고 있다"**는 비전을 심어줍니다.
